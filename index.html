How to Safely Delete NodeJS Project or Dependencies

Web Development with the Node ecosystem can be an engaging process that allows us to experiment with new tools at a moment's notice. However, the best experimentation is done so with an understanding of safety precautions and the ability to reset things if something goes wrong.

The Node development process starts by typing the npm install command into the command-line interface. This instruction triggers a series of actions where all of the "dependencies" and "devDependencies" listed in the package.json file are retrieved then installed seamlessly in a layered folder structure that can be difficult to undo if you attempt to delete the items manually. Windows users sometimes get an error that says "source path too long" when trying to remove certain items because the Windows operating system is typically limited to file path locations up to 260 characters long.

This tutorial will show you how to uninstall/delete an entire Node Project or singular dependency with three quick steps.

Step 1 - Edit the package.json file

Locate the package.json file in the root of the project folder. Eliminate all the dependencies and/or devDependencies you wish to get rid of. Remove them all if you wish to delete the entire project. Save the file.

                
                    {
                        "name": "cool-project",
                        "version": "1.0.0",
                        "description": "awesome modern cool project",
                        "main": "index.js",
                        "scripts": {},
                        "author": "Harry Belafonte",
                        "license": "ISC",
                        "devDependencies": {},
                        "dependencies": {}
                    }

Step 2 - Use the npm prune command

Make sure your command-line tool is currently inside the folder/directory of the project you wish to change.

Type and enter the npm prune command. Execution of this command without the addition of any specific files or flags will remove all of the packages not included in the package.json file. This step may take a while depending upon the power of your individual computer.

                
                    change directory into the project folder

                    cd project-path/my-cool-project

                    .... 

                    npm prune

Step 3 - Delete the Project

IMPORTANT: If you only wish to remove/uninstall one or more dependencies you can stop at the previous step. Continue if you wish to remove the entire project from your computer.

The npm prune command has now successfully stripped unwanted projects from the node module folder. You may now delete the folder from your computer as you would do to any other file/folder normally.

The npm prune command is one of the most powerful Node commands for developers because it offers the opportunity to start things over with a clean slate and avoid the bloat of having useless files saved on your computer.
